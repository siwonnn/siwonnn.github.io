[Attend First, Consolidate Later: On the Importance of Attention in Different LLM Layers](https://arxiv.org/abs/2409.03621)

### The claim I tested
This paper suggests that LLMs have a two-phase processing where the first part gathers informations, and another part consolidates that information.
To support this claim, they consider manipulations such as inserting noise in hidden states, or freezing them.
In this post, I test the claim that freezing the model at layer k and copying the hidden state at that layer to all continuing layers does not lead to significant performance drop.

### What freezing means
Freezing means to stop updating hidden states after k layers. The model does the attention and mlp calculation up to layer k, then we use the same hidden states after on, except for the last token. Each token's hidden state needs to encode everything relevant about this position for downstream use. However, at the last position, it has to produce a probability distribution of the next token, which is more specific job than being readable. This is why the last token is special, and because of this we continue the computation of hidden state for the last token.

### What I did
[code: implement freezing and capitals task experiment](https://github.com/siwonnn/attend-first-consolidate-later/commit/9aec542b4d0a42398ae7b89e3182d160d0722300)

I set up a Llama-3.2-1b model for experiment. Using pytorch's `register_forward_pre_hook` function, I registered a hook on forward pass to freeze the hidden state after kth layer. For evaluation, I used a country-capital city dataset as used on the paper. The data is collected from [country-json](https://github.com/samayo/country-json), and I filtered datas where the capital city is a single token, leading to 57 data points. Then I ran the prediction for the capital city, given the country with the unmanipulated model and the manipulated model each. The prompt gives a one-shot example to only generate the city name without explanations.

### The result
![Result Plot](/img/afcl_freeze_plot.png)
This plot resembles that on the paper, shown in Figure 2. The curve shows that the model achieves zero accuracy till k=4, and it increases sharply from k=4 to k=7. Then it reaches the baseline accuracy from k=14. This shows a two-phase processing suggested on the paper. When the model is manipulated from the bottom layer, the accuracy is very low since the phase where it gathers information has been interrupted. But when the freezing is applied on the top part of the layers, it does not significantly undermine the performance since it is the phase where it's consolidating previously gathered informations. Computation in the top layers matters less than in the bottom layers.

### Thoughts and further work
Through loading the model, inspecting its structure, and checking how the shape of matrices match, I was able to better understand for myself how transformers work internally. This was my first time implementing a paper on my own, and it feels like a good first step after learning the concepts of transformers. While doing this work, a few questions came to mind:

1. Does this hold in another language? Would the critical layer differ by langauge? Korean, for instance, tokenizes very differently from English, so the information-gathering phase might behave differently.
2. Could we schedule attention and MLP computation by layer? If information gathering mainly happens in the bottom layers, what if we computed only attention in that phase and only the MLP in the top layers? Could this cut computation cost while keeping performance?

I plan to continue with these questions, and to implement the other manipulations from the paper.
